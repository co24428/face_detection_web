Benchmark

Quick background about the metrics and how to read them. Values vary by device, browser, and settings.

What we measure

  • FPS (render): how smooth the UI looks.
  • Throughput (TPS): inferences per second (model loop).
  • Latency: end-to-end (E2E), model-only (Run), and pre/post steps.
  • Stability: how latency changes over time (p50/p90 drift).
  • Resources: dropped frames, memory hints.
  • Accuracy (optional): AP@0.5 with a small labeled set.

Key metrics

  • FPS: frames rendered per second (UI smoothness).
  • Throughput (TPS): inferences/sec; can differ from FPS.
  • Latency (ms):
    • E2E: capture → boxes painted.
    • Run: session.run() only.
    • Pre/Post: resize/normalize & decoding/NMS/draw.
  • p50 / p90 / avg: distribution snapshots; p90 is a good “worst-case feel”.
  • AP@0.5 (optional): IoU ≥ 0.5 matching with a small GT set.
